# Zijun Method Development

This directory is the isolated workspace for the Task 2 method-development part:
predicting LLM pretraining loss curves across learning-rate schedules.

The working assumption is that the two baseline reproductions already exist:

- Tissue et al. momentum-law style baseline: `code/momentum.py`
- Luo et al. multi-power-law baseline: `code/MultiPowerLaw/`

This folder should contain only Zijun's new fitting method, analysis scripts, and
derived outputs. Keeping this separate makes it easier to compare against the two
baselines and avoids editing imported reproduction code.

## Protocol

Use the course-provided curves in:

```text
loss curves/gpt_loss+lrs.pkl
```

The expected evaluation protocol is:

1. Fit the new method on the cosine LRS curve.
2. Predict the WSD LRS curve.
3. Optionally also report the 8-1-1 LRS curve as an auxiliary test.
4. Compare metrics against the reproduced Tissue and MultiPowerLaw baselines.

Recommended metrics:

- MAE
- RMSE
- MAPE
- R2

## Layout

```text
zijun/method_development/
├── README.md
├── baseline_results/      # summaries copied from reproduced baselines
├── figures/               # plots generated by this method
├── outputs/               # metrics, predictions, fitted parameters
├── scripts/               # runnable experiment entry points
└── src/                   # reusable implementation
```

## Current Starting Point

`scripts/run_feature_fit.py` is a lightweight starting method. It fits a ridge
model on hand-built schedule features from the cosine curve and evaluates on WSD
and 8-1-1. This is intentionally simple so it can serve as a controllable
starting point for developing a more principled fitting method.

Run it from the repository root:

```bash
python zijun/method_development/scripts/run_feature_fit.py
```

It writes:

```text
zijun/method_development/outputs/feature_fit_metrics.csv
zijun/method_development/outputs/feature_fit_predictions.csv
zijun/method_development/figures/feature_fit_predictions.png
```

## Method Ideas To Explore

- Add schedule-aware features motivated by the momentum-law baseline.
- Fit in log-loss space but evaluate in raw-loss space.
- Use robust losses or weighted fitting to reduce sensitivity to noisy early
  steps.
- Use residual correction on top of a reproduced baseline rather than fitting
  from scratch.
- Compare extrapolation quality across the decay region rather than only full
  trajectory average error.
